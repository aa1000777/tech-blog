📱 AndroidScreenMirror

📋 项目信息

📁 Android 📱 投屏 🔧 维护中
安卓屏幕镜像项目，支持WiFi P2P/TCP/UDP，实现手机到Pad的实时投屏和触摸反控功能。

📂 核心文件结构

app/src/main/java/com/example/screenmirror/
├── MainActivity.java          # 主界面，选择设备角色
├── SenderActivity.java        # 发送端（采集+编码+传输）
├── ReceiverActivity.java      # 接收端（接收+解码+渲染）
├── ScreenCaptureService.java  # 屏幕录制服务
├── WifiP2pService.java      # WiFi P2P服务
├── NetworkService.java       # 网络传输服务
├── Constants.java            # 常量定义
├── RtpPacketizer.java        # RTP打包器
├── LowLatencyNetwork.java    # 低延迟网络传输核心
└── TouchEventSerializer.java # 触摸事件序列化

⚙️ 核心技术

🎯 屏幕采集

MediaProjection API + VirtualDisplay

🎬 视频编码

MediaCodec H.264硬编码，低延迟优化

🌐 网络传输

TCP/UDP/WiFi P2P 三种模式，支持优先级队列

👆 触摸反控

TouchEventSerializer 紧凑二进制协议

📖 详细文档

功能特性

  1. 屏幕采集：使用MediaProjection API捕获屏幕内容
  2. 视频编码：使用MediaCodec进行H.264硬编码
  3. 网络传输：支持TCP和UDP协议传输视频流
  4. 解码渲染：接收端解码并显示视频流
  5. 触摸反控：Pad端触摸控制手机端操作
  6. 双连接模式：WiFi P2P直连和局域网连接

低延迟优化

  • 优先级队列：触摸事件 > 关键帧 > 普通帧
  • RTP打包：单包/FU-A分片模式
  • 前向纠错(FEC)：20%冗余抗丢包
  • Android Q+ 低延迟编码模式 (KEY_LATENCY)

编译要求

  • Android Studio 2022.3.1+
  • Android SDK 33+
  • Java 8+
  • 两台Android 5.0+ 设备
