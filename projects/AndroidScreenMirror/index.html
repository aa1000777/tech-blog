📦 SDK 模块结构（screenshare-sdk）

screenshare-sdk/src/main/java/com/screenshare/sdk/
├── ScreenshareSDK.java         # SDK 唯一入口类
├── Sender.java                # 发送端（状态机）
├── Receiver.java              # 接收端（状态机）
├── SenderConfig.java          # 发送端配置
├── ReceiverConfig.java        # 接收端配置
├── Common/
│   ├── AtomicBuffer.java      # 原子化预分配缓冲区（GC优化）
│   ├── ErrorCode.java         # 错误码体系（1xxx-5xxx）
│   ├── SenderState.java        # 发送端状态机
│   └── ReceiverState.java      # 接收端状态机
├── capture/
│   └── ScreenCapturer.java    # 屏幕采集（MediaProjection）
├── codec/
│   ├── VideoEncoder.java      # 硬件 H.264/H.265 编码
│   ├── VideoDecoder.java      # 硬件 H.264/H.265 解码
│   └── AudioDecoder.java      # 音频解码（占位符）
├── network/
│   ├── UdpChannel.java        # UDP 通道
│   └── RtpSession.java        # RTP 打包/解包
├── service/
│   └── ScreenCaptureService.java  # 前台服务
├── touch/
│   ├── TouchEncoder.java      # 触摸编码（独立通道）
│   ├── TouchDecoder.java      # 触摸解码
│   └── TouchInjectorService.java  # AccessibilityService 注入
└── wifi/
    ├── P2pConnectionManager.java   # WiFi P2P 管理
    └── P2pBroadcastReceiver.java   # P2P 广播接收

🎯 核心技术指标

⚡ 延迟目标

视频延迟 <50ms | 触摸延迟 <10ms

🔧 性能优化

原子缓冲区消除 GC | 独立触摸通道 | RTP FU-A 分片

📡 连接模式

WiFi P2P 直连 → WiFi 局域网 TCP 降级

🎬 编码

H.264/H.265 硬件编码（MediaCodec）+ KEY_LATENCY=0
