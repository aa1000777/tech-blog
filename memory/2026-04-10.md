## 2026-04-10 VPN 端口修改

**用户请求**：将VPN端口从5333改为5334并重启VPN

**执行操作**：
1. 备份原配置文件：`/etc/wireguard/wg0.conf.backup.20260410_2249xx`
2. 修改 `/etc/wireguard/wg0.conf` 中的 `ListenPort = 5333` 为 `ListenPort = 5334`
3. 重启 WireGuard 服务：
   ```bash
   systemctl stop wg-quick@wg0
   systemctl start wg-quick@wg0
   ```
4. 验证服务状态：`wg show wg0` 显示 listening port: 5334 ✅

**服务器端检查**：
- WireGuard 服务正在监听 UDP 5334 端口
- 需要用户检查云服务器安全组规则，确保 UDP 5334 端口已开放
- 客户端需要更新配置文件中的 Endpoint 端口号

**可能原因**：原5333端口可能被网络运营商或防火墙封锁，更换为5334可能解决连接问题。

**下一步**：
1. 用户需登录云控制台（腾讯云/阿里云）修改安全组规则，开放 UDP 5334 端口
2. 用户需在客户端设备上更新 WireGuard 配置文件，将 `Endpoint = 服务器IP:5333` 改为 `Endpoint = 服务器IP:5334`
3. 重新连接测试

**学习点**：VPN 端口封锁是常见问题，可尝试更换为其他非标准端口（如 53, 443, 5353, 5334 等）绕过封锁。
