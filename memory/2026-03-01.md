## VPN问题深度解决和学习

### 问题背景
用户需要WireGuard VPN连接，但遇到多个复杂问题：
1. 初始连接失败
2. 云服务器端口未开放
3. Windows网络层问题
4. 分流配置需求

### 学到的关键知识

#### 1. WireGuard核心配置
- **密钥管理**：公钥/私钥必须匹配，不同设备需要不同密钥
- **端口选择**：UDP协议，常用端口53/443/51820，云服务器需手动开放
- **路由控制**：`AllowedIPs = 0.0.0.0/0` 控制所有流量走VPN
- **连接保持**：`PersistentKeepalive` 防止连接断开
- **MTU优化**：1280/1400/1420/1500 测试最佳值
- **DNS设置**：Cloudflare(1.1.1.1)通常最快

#### 2. 问题诊断流程
1. **基础连通性**：`ping 10.0.0.1`
2. **路由检查**：`route print | findstr "0.0.0.0"`
3. **服务状态**：WireGuard服务是否运行
4. **防火墙检查**：Windows防火墙 + 云服务器安全组
5. **端口测试**：UDP端口连通性

#### 3. 特定问题解决方案
- **"一般故障"错误**：网络重置 + 重启电脑
- **云服务器端口不通**：腾讯云安全组添加UDP规则
- **只能访问内网**：检查`AllowedIPs`和iptables NAT规则
- **速度慢**：优化MTU、DNS、TCP参数

#### 4. 分流策略学习
- **国内直连/国外走VPN**：通过路由表或`AllowedIPs`实现
- **WireGuard限制**：配置复杂，IP段太多
- **替代方案**：Clash、V2Ray等专业分流工具

### 最终解决方案
1. **服务器配置**：UDP 53端口，优化iptables规则
2. **PC配置**：MTU 1400，Cloudflare DNS，`AllowedIPs = 0.0.0.0/0`
3. **监控设置**：仅流量统计，无访问日志
4. **分流准备**：提供路由脚本和配置方案

### 学习收获
1. **系统性排查**：从基础到复杂，逐步诊断
2. **平台差异**：Windows/云服务器的特殊问题
3. **配置细节**：每个参数的影响和优化
4. **用户需求理解**：隐私保护（无日志）、速度优化、分流需求
