## VPN端口封锁与解决方案（2026-04-10）
通过今天的端口修改请求，学习了端口封锁的应对策略：

### 1. 常见封锁端口
- 标准VPN端口（51820, 1194, 1701）容易被识别封锁
- 运营商可能封锁特定UDP端口范围
- 公共WiFi/企业网络常封锁非标准端口

### 2. 规避策略
- **使用非标准端口**：如53(DNS), 443(HTTPS), 5353(mDNS), 5334等
- **端口随机化**：定期更换端口减少被封锁风险
- **协议混淆**：WireGuard over Websocket等（需要额外配置）

### 3. 操作流程
1. 修改服务器端 `/etc/wireguard/wg0.conf` 的 `ListenPort`
2. 重启WireGuard服务：`systemctl restart wg-quick@wg0`
3. 检查监听状态：`wg show wg0`
4. 更新云服务器安全组规则（UDP新端口）
5. 客户端更新 `Endpoint` 端口并重新连接

### 4. 诊断技巧
- 使用 `nc -u -vz 服务器IP 端口` 测试UDP端口连通性
- 检查 `wg show` 的握手时间，判断是否被中间设备干扰
- 对比不同网络环境（4G vs WiFi）的连接情况

### 5. 最佳实践
- 记录每次端口变更，保持服务器-客户端同步
- 保留常用端口备选列表（53, 443, 8080, 5353, 5333, 5334, 5335等）
- 设置监控告警，及时发现端口封锁
